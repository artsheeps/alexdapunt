SumUp Onboarding

The following interaction design focusses on the onboarding process for the SumUp app. It should inform the user in regards of the product and highlights the benefits of mobile payments.

Common Examples

Regarding the onboarding process for users, we usually think of task tutorials to introduce the functionality of the particular app. This is not always the ideal way to successfully adopt the user to the product. It is important to get the user started with the app and teach him on-the-fly whilst using it.

The Concept

With the first teaser we want to welcome the user of using the SumUp app for mobile payments. This should already be enough to get a rough idea of what the product has to offer. Even if we add two additional information slides we give the customer the possibility to sign up directly without having to browse through the slides first.

After the user has signed up, he will have direct access to the app, where he can explore and get a grip on the functionality and features on offer. Once opened the user will have a notification pending to inform about the further process for activating the card reader or how to purchase the particular device.

Additionally, the app should have certain functionality without having the card-reader installed yet in order to keep the user entertained in the app before he considers buying the hardware needed (e.g. Sending invoices).
